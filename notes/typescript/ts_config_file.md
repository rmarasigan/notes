# TS â€“ tsconfig.json

The `tsconfig.json` file is a file which is in JSON format. In the `tsconfig.json` file, we can specify various options which tell the computer how to compile the current project. We can compile a typescript project in one of the following ways:
* **By invoking `tsc` with no input files**
   * In this case, the compiler searches the `tsconfig.json` file starting in the current directory following the parent directory.
* **By invoking `tsc` with no input files and a `--project` or just `-p` command**
   * The compiler specifies the path of a directory which contains a `tsconfig.json` file. It also specifies a path to a valid `.json` file which contains the configurations.

The `tsconfig.json` is generally put in the root folder of the project and it provides the compiler options required to compile the project.

## Create `tsconfig.json`
Navigate to the root folder of your project in your terminal or command prompt or powershell and run the following command:

```bash
dev@dev:~$ tsc --init
Created a new tsconfig.json with:                                                                                       
                                                                                                                     TS 
  target: es2016
  module: commonjs
  strict: true
  esModuleInterop: true
  skipLibCheck: true
  forceConsistentCasingInFileNames: true

You can learn more at https://aka.ms/tsconfig.json
```

## `include` and `exclude` Properties
An **include** and **exclude** properties allows us to take an array pattern to add or remove a list of TypeScript files from the compilation process. The **`include`** and **`exclude`** properties take a list of glob-like patterns. The supported glob wildcards are:
* `*` matches zero or more characters (excluding directory separators)
* `?` matches any one character (excluding directory separators)
* `**/` recursively matches any subdirectory

If we have a large number of files, we use the `include` property, which allows us to take an array pattern to add a list of `.ts` files to the compilation process.
```json
  "include": ["src/**", "tests/**"]
```

We can exclude files in the same way with the `exclude` property.
```json
  "exclude": ["node_modules", "**/*.spec.ts"]
```

## `compilerOptions` Property
We can customize the compiler options properties by using the `compilerOptions`. It allows specifying additional options to the TypeScript compiler. It can be omitted, in which case the compiler's default are used. Here's a sample JSON file describing how you can define a `tsconfig.json` file containing different parameters of the `compilerOptions` property.
```json
{
   "compilerOptions": {
      "module": "system",
      "noImplicitAny": true,
      "removeComments": true,
      "allowUnreachableCode": false,
      "strictNullChecks": true,
      "outFile": "../javascript/type_script/js_file_name.js",
      "sourceMap": true
   }
}
```

## `compileOnSave`
It is the property which is used to set the IDE to compile the included TypeScript files and generate the output automatically. It signals to the IDE to generate all files for a given `tsconfig,json` upon saving.

```json
{
   "compileOnSave": true,
   "compilerOptions": {
      "noImplicitAny" : true  
   }
}
```
