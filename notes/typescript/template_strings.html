    ECMA 6 ‚Äì Template Strings¬∂

    Template literals (template strings) are literals delimited with backtick (`), allowing for multi-line strings, for string interpolation with embedded expressions, and for special constructs called tagged templates. Template strings allow you to embed code expressions within string literals.

    Without template strings¬∂

    index.html¬∂

    <!DOCTYPE html>
<html lang="en">
<head>
   <meta charset = "UTF-8">
   <meta http-equiv = "X-UA-Compatible" content = "IE=edge">
   <meta name = "viewport" content = "width=device-width, initial-scale=1.0">
   <title>
      TypeScript Course
   </title>
</head>
<body>
   <script src = "script.js"></script>
</body>
</html>

   document.write("Name: " + name);

   document.write("Profession: " + profession);
   document.write("Gender: " + gender);

    document.write("Name: " + name);

    With template strings¬∂

    script.js¬∂

    function print(name, age, profession, gender)
{
   document.write(`Name: ${name}
                  Age: ${age}
                  Profession: ${profession}
                  Gender: ${gender}`);
}

print("Robert", 29, "Developer", "male");

    Output:

    Name: Robert Age: 29 Profession: Developer Gender: male

    As you can see, it got all the spaces in between. If you add a <br>, it will go create a new line.

    document.write(`Name: ${name} <br>
               Age: ${age} <br>
               Profession: ${profession} <br>
               Gender: ${gender} <br>`);

    Output:

    Name: Robert
Age: 29
Profession: Developer
Gender: male
